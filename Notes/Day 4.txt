Why Cookies & Sessions needed?
	HTTP is stateless.
	Which means:
		Request 1 => Server forgets everything.
		Request 2 => Server forgets everything once again.

Concept			Cookies						Sessions
Stored on		Client(browser)				Server
Size			Small(not more that 4KB)	Large
Security		Less Secure					More secure
Data Type		Plain data					Any JS Object
Common Use		Preferences					Authentication

What is a Cookie?
	A cookie is a small key-value pair stored in the user's browser.
		e.g. user=James; theme=dark;
What is a Session?
	A session stores user data on the server, and only a session ID is stored in the browser cookie.
	
	Client → cookie: sessionId
	Server → session data
