Introduction to GEN AI
What is GEN AI?
	GEN AI stands for Generative AI and refers to a type of artificial intelligence that is capable of generating new content-such as text, images, videos, music, code, etc. rather than just analyzing or classifying existing data.
	It works by learning patterns from large amounts of data & then using those patterns to generate original outputs that resemble what humans create.
	GEN AI doesn't just understand data, ir creates new content from it.

	• Microsoft Copilot
		What is it?
			Enterprise-focused GEN AI assistant that is integrated into Microsoft 365 apps.
			Uses GEN AI to create, summarize, analyze & transform the content like documents, emails, presentations & data.
			Designed to boost productivity & decision making using both AI models & your organizaion's data while respecting security & permissions compliances.
			Think of it as GEN AI for everyday business work.

	• GitHub Copilot
		What is it?
			Developer-focused GEN AI assistant embedded in code editors(VS Code, Visual Studio, JetBrains, Eclipse, etc.).
			Uses GEN AI to write, complete, explain, & refactor code based on the context & the comments.
			Trained on large amounts of public code & optimized for software development workflows.
			Think of it as GEN AI for coding & software engineering.

	• How Copilot works — LLMs, context window, tokens
		LLMs-The Enginw
			It powers all AI including MS65 Copilot, GitHub Copilot.
			What an LLM is?
				It stands for Large Language Model and technically a neural network trained on massive volumes of text & code to learn statistical patterns such as:
					Syntax
					Naming Conventions
					API usageCommon design Patterns
					Natural Language -> code mappings
				LLMs don't reason like humans & don't understand meaning in the semantic sense.
				They predict:
					What token is most likely to come next given the previous token?
				This applies equally to:
					English paragraphs
					C# methods
					YAML pipelines
					SQL Queries
					and many more.
		Token: The Atomic Unit
			What is token?
				A token is not necessarily a word.
				A token can be:
					A word fragment
					A symbol
					Part of keyword
					Withespace or punctuation
				e.g.
					"public" -> token
					"authentication" - May be 2-3 tokens
					"OrderServiceFactory" -> multiple tokens
					"{", "(", "=>" -> separate tokens
				LLMs operate only on tokens.
			Why tokens matter?
				They matter as they determine:
					How much information fits into the model's "memory".
					Cost & Latency
					Whether Copilot "forgets" earlier parts of your code.
		Context Window: Copilot's short term memory
			What is context Window?
				The context Window is the maximum number of tokens Copilot can see at once.
				It typically includes:
					Current file content
					Nearby code blocks
					Open editor tabs
					Your cusrsor position
					Your comment or prompt
					Language/framework signals
			Copilot does not see:
				Your entire repository
				Closed Files
				Jira Tickets
				Architectural Diagrams
				Production Configs
				Runtime behvior
				
				// Create a validator for this request
				public record CreateOrderRequest(Guid CustomerId, decimal Total);
		Prompt + Context -> Prediction
			Whenever you type a comment, Copoilot performs the following steps:
				Tokenizes
					Your Code
					Your comment
					Nearby Context
				Feeds tokens into the LLM
				Calculate probabilitis for the next tokens.
				Propses the most likely continuation
				e.g. services.AddScoped<IOrderService, OrderService>();
			What Copilot doesn't remember?
				Copilot does not have: 
					Long term memory
					Awareness of previous chats
					Persistent understanding of your project
			Practical Engineering Implications
				To get better results:
					Keep related code nearby
					Use clear comments
					Name things explicitly
					Open relvant files
					Avoid huge, unfocused files.
				To avoid risk:
					Review every suggestion
					Don't trust security-sensitive code
					Validate APIs & behavior
					Treat Copilot output as draft, not decision.
